## 缓存计费：这个功能很值得关注

DuckCoding 支持**缓存计费**，在 Claude Code 的实际使用场景里，这个特性相当实用。

Claude Code 工作时会反复读取同一个项目的文件——代码上下文、同一个函数、同一段注释——这些内容命中缓存后就不需要重复计费。实际使用下来，有用户反馈在命中率高的场景里，实际消耗的额度比预期少很多。DeepSeek 模型的缓存命中率据说也不低，对于需要反复跑相似请求的场景，实际成本能降不少。

## 快速上手：配置其实很简单

以 Claude Code 为例，核心就三步：

1. 注册账户，充值，在控制台创建「Claude Code 专用分组」的 API Token
2. 修改项目目录下的 `.claude/settings.json`，填入 Token 和中转地址：
json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "你的Token",
    "ANTHROPIC_BASE_URL": "https://api.duckcoding.ai"
  }
}

3. 进入项目目录，运行 `claude`，就能用了

CodeX 和 Gemini CLI 配置方式类似，都有专用分组 Token，别搞混了。
